## Install

```bash
npm i prop-for-that
```

## Three usage modes

### 1. Auto (zero-config)

Import `prop-for-that/auto` and it does everything for you:

- Attaches the default global sources (`viewport`, `pointer`) to `:root`.
- Scans the DOM for `[data-prop]` attributes and binds the named sources to those elements.
- Watches for DOM changes with a `MutationObserver` so dynamically added elements are handled automatically.

```html
<script type="module">import 'prop-for-that/auto'</script>

<input type="range" min="0" max="100" data-prop="range">
<section data-prop="size visibility">…</section>
```

```css
input {
  background: hsl(calc(var(--live-value-pct) * 120) 80% 50%);
}

section {
  opacity: var(--live-visible-ratio);
}
```

### 2. Imperative

Import individual functions for explicit control over which sources are active and when they're torn down:

```js
import { global, bind, configure } from 'prop-for-that'

// Write viewport/pointer to :root
global(['viewport', 'pointer'])

// Write range source to a specific element; returns a disposer
const slider = document.querySelector('input[type="range"]')
const dispose = bind(slider, ['range'])

// Later — tear it down cleanly
dispose()
```

Use `configure()` to change the property prefixes or the global root target before attaching any sources:

```js
import { configure } from 'prop-for-that'

configure({ livePrefix: '--pft-', constPrefix: '--pft-const-' })
```

### 3. Head constants

Some values only need to be read once and are critical for the first paint (scrollbar width, device pixel ratio, CPU cores). Load the head entry as a synchronous inline script so the properties exist before any CSS is applied:

```html
<head>
  <script type="module">import 'prop-for-that/head'</script>
  <!-- sets --const-scrollbar-w, --const-dpr, --const-cores, and more -->
</head>
```

These are written with `--const-*` names and are never overwritten after that initial write.
