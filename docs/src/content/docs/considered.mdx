Rides the writer (lands post-paint); read the same value in a `<head>` script for a pre-paint constant |
| [`page-lifecycle`](https://github.com/argyleink/prop-for-that/issues?q=is%3Aopen+label%3Aproposal+page-lifecycle) | the suspension layer beneath `page-visible`: `--const-bfcache-restored` (latches when the page was restored from the back/forward cache, via `pageshow.persisted`) and `--live-frozen` (Page Lifecycle `freeze`/`resume`) |
| [`user-activation`](https://github.com/argyleink/prop-for-that/issues?q=is%3Aopen+label%3Aproposal+user-activation) | `--live-user-active` from `navigator.userActivation.isActive` (transient activation) ŌĆö the *live* "interacting right now" complement to the sticky `interacted` latch, for gating autoplay / audio-unlock windows |

_Why not `pageswap` / `pagereveal`? Those cross-document [View Transition](https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API) hooks fire **before the first rendering opportunity** and exist to configure or skip a transition imperatively ŌĆö the opposite of this library's read-state-then-rAF-flush model, which writes **after** paint. The one durable thing worth reading at that moment, how the user arrived, is surfaced synchronously instead via the `navigation` const above._

## ­¤Ü½ Out of scope

CSS or the platform already does these well, so prop-for-that won't duplicate them:

- **Interaction states:** `:hover`, `:focus`, `:focus-within`, `:focus-visible`, `:active`, `:checked`, `:disabled`, `:required`, `:placeholder-shown`, `:empty`, `:has()`, `[open]`, `:modal`, `:fullscreen`, `:popover-open`.
- **Preferences:** `prefers-color-scheme`, `prefers-reduced-motion`, `prefers-reduced-data`, `prefers-contrast`, `forced-colors`, `inverted-colors`.
- **Layout queries:** media / container / `@container style()` queries; `orientation`, `resolution`, `aspect-ratio` MQ.
- **Units:** `vw/vh/dvh/svh/lvh/vmin/vmax`, `cqw/cqh`, `env()` safe-area insets.
- **Scroll-linked:** scroll position/progress ŌåÆ `animation-timeline: scroll()` / `view()` (compositor-driven, no library needed).
