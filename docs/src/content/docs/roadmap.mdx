## v0.1 — Core spine

- Six core sources: `viewport`, `scroll`, `pointer`, `size`, `visibility`, `range`
- `auto` entry with default globals and `[data-prop]` scanning
- `head` entry for FOUC-safe constants
- Browser naming test: `setProperty` ↔ `var()` round-trip across Chromium, Firefox, and WebKit

## v0.2 — Plugins

Opt-in exotic sources:

- **Global**: `scroll-velocity`, `network`, `battery`, `online`, `fps`, `clock`, `orientation`, `motion`, `light`, `visual-viewport`
- **Element**: `el-scroll`, `pointer-local`, `media`, `field`, `children`, `caret`

## v0.3 — Docs & animatable values

- This documentation site with live interactive demos
- Optional `@property` opt-in for registered properties so values become animatable via CSS `transition` and `animation`

## v1.0 — Stable

- Stable public API
- Full plugin catalog
- Framework adapter notes (React, Vue, Svelte, Solid)
- Performance benchmark suite

---

## Open questions

These design decisions are still being considered:

1. **Default global set** — should `auto` attach `viewport scroll pointer` by default, or a leaner/heavier default?
2. **Prefix strings** — keep `--live-`/`--const-`, or namespace harder (`--pft-live-`) to avoid collisions with user-defined properties?
3. **Package name** — `prop-for-that` as-is, or a scoped package (`@argyle/prop-for-that`)?
