The `pointer` source writes the cursor's position to `:root`: `--live-pointer-x` / `-y` in pixels, and `--live-pointer-x-ratio` / `-y-ratio` as 0–1 fractions of the viewport. Move your pointer and watch the spotlight, the dot, and the readout follow, all in CSS.

<PointerDemo />

## The wiring

`pointer` is one of auto's default global sources, so the import is the whole setup — then CSS does the rest:

```html
<script type="module">import 'prop-for-that/auto'</script>
```

```css
.spotlight {
  background: radial-gradient(
    9rem 9rem at
    calc(var(--live-pointer-x-ratio) * 100%)
    calc(var(--live-pointer-y-ratio) * 100%),
    var(--accent), transparent 70%
  );
}
```

## Scoped to an element

The global source answers "where is the pointer in the _viewport_?" The
`pointer-local` plugin answers "where is it _within this element_?" It writes
`--live-local-pointer-x-ratio` / `--live-local-pointer-y-ratio` (0–1 across the bound element's box) and
`--live-local-pointer-inside` (`1` while the pointer is over it) to that element only,
so its descendants inherit them and nothing outside does.

<PointerLocalDemo />

It's a plugin, so register it once **before the DOM is scanned** (auto registers only the core sources); after that the element opts in with `data-props-for`:

```html
<script type="module">
  import 'prop-for-that/auto'
  import { register } from 'prop-for-that'
  import { pointerLocal } from 'prop-for-that/plugins'
  register(pointerLocal)
</script>

<div data-props-for="pointer-local">…</div>
```

```css
.box {
  /* spotlight tracks the pointer within the box, lit only while it's inside */
  background: radial-gradient(
    7rem 7rem at calc(var(--live-local-pointer-x-ratio) * 100%) calc(var(--live-local-pointer-y-ratio) * 100%),
    color-mix(in oklab, var(--accent) calc(var(--live-local-pointer-inside) * 50%), transparent),
    transparent 70%
  );
}
```
