CSS can't read an image's pixels, so it can't tint a card, scrim, or caption to match the artwork inside it. The `img-color` plugin extracts a small **palette** from an `<img>` — the dominant color plus an accent, a darkest, a lightest, an average, and a warm/cool reading — and writes each as channels you compose with `rgb()`, plus a luminance you branch on to keep text legible.

It's the heavier sibling of the [`img`](/reference/plugins/#element-plugins) plugin (which does load state and natural size) — kept separate so the pixel-reading cost is opt-in.

Each card above is themed from its own artwork: the caption is painted with the **dominant** color, the glow is the **accent**, the chips show the full palette, and the bar reads its **temperature** — all in CSS, with text flipping black or white from each color's luminance.

## The props

Written on the bound element on the [`live`](/concepts/naming-and-cadence/) cadence, recomputed whenever the image loads (so a `src` swap updates them). Every swatch shares the same shape: `r` / `g` / `b` channels (`0`–`255`) and an `l` luminance (`0`–`1`, Rec. 709 weights — threshold it for light/dark text).

| Property | What it is |
| --- | --- |
| `--live-img-r`, `-g`, `-b`, `--live-img-l` | the **dominant** color (the most common one) |
| `--live-img-accent-r`, `-g`, `-b`, `-l` | the **most vibrant** color — a punchy, mid-toned accent (reuses the dominant for a grayscale image) |
| `--live-img-dark-r`, `-g`, `-b`, `-l` | the **darkest** color that isn't black |
| `--live-img-light-r`, `-g`, `-b`, `-l` | the **lightest** color that isn't white |
| `--live-img-avg-r`, `-g`, `-b`, `-l` | the **average** (mean of every pixel) — distinct from the dominant (the mode) |
| `--live-img-temp` | the image's **temperature**, `−1` (cool) … `+1` (warm), from its red-vs-blue balance |

```css
.card { background: rgb(var(--live-img-r) var(--live-img-g) var(--live-img-b)); }
.tag  { background: rgb(var(--live-img-accent-r) var(--live-img-accent-g) var(--live-img-accent-b)); }

/* contrasting text: round() snaps luminance to 0 or 1 → white or black */
.card { color: hsl(0 0% calc((1 - round(var(--live-img-l), 1)) * 100%)); }

/* warmer images lean the UI warmer */
.frame { --hue: calc(220deg - var(--live-img-temp) * 200deg); border-color: hsl(var(--hue) 60% 50%); }
```

## How it stays cheap

The whole palette is found by **downsampling**, not by scanning every pixel — and every swatch comes from a **single pass**:

1. `createImageBitmap(img, { resizeWidth: 16, resizeHeight: 16 })` decodes and shrinks the image **off the main thread**.
2. It's drawn to a 16×16 `OffscreenCanvas` (no DOM node), and those 256 pixels are read back once.
3. One loop buckets the colors into a coarse grid and tallies the pixel sum; the dominant is the most-populated bucket, the accent/dark/light fall out of the same buckets by saturation and luminance, and the average and temperature come from the running sum. Transparent pixels are skipped.

Both `OffscreenCanvas` and `createImageBitmap` are feature-detected, with a plain `<canvas>` / direct-draw fallback, so it runs wherever canvas does and no-ops where it doesn't.

:::caution[Cross-origin images need CORS]
Reading pixels from a cross-origin image taints the canvas and throws. For images from another origin, add `crossorigin="anonymous"` **and** serve them with permissive CORS headers. Without that the plugin simply **writes nothing** — so always give `var()` a fallback: `rgb(var(--live-img-r, 128) …)`. Same-origin and `data:` images (like this demo's) just work.
:::

## The wiring

```html
<script type="module">
  import 'prop-for-that/auto'
  import { registerPlugins, imgColor } from 'prop-for-that/plugins'
  registerPlugins(imgColor)
</script>

<!-- bind the figure so the caption (a sibling of the img) inherits the color -->
<figure data-props-for="img-color">
  <img src="/cover.jpg" alt="">
  <figcaption>…</figcaption>
</figure>
```

```js
import { propsFor, register } from 'prop-for-that'
import { imgColor } from 'prop-for-that/plugins'

register(imgColor)
propsFor(document.querySelector('figure'), ['img-color'])
```
