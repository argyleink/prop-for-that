prop-for-that

JavaScript does the minimum.
CSS does the reacting.

Runtime state JS knows but CSS can't see — pointer, scroll, size, visibility, slider values — written into batched --live-* custom properties. Every effect below is plain CSS reading a var(). No JS reads values. No JS sets styles in the animation path.

Slider-driven gauge

One <input type="range">. Its --live-value-pct drives the gauge sweep, the hue, the dot's size, and the printed number — entirely in CSS, including the digits via a counter.

Three demos, one idea: expose state, then get out of CSS's way. Open DevTools and watch --live-* change on :root and the bound elements.
