Scroll storytelling

The progress bar up top is a native CSS scroll-driven animation — animation-timeline: scroll(), on the compositor, no library. prop-for-that handles the part CSS can’t: each panel fades and lifts the moment it’s scrolled into view, reading its own --live-visible-ratio (0 → 1).

03 / range

One value, many readers

Bind the range source to the container. It finds the inner <input> but writes --live-value and --live-value-pct on the container — so the gauge ring, the counted number, and the slider itself all read the same value (custom properties inherit down). Drag or press ← →.

Three demos, one idea: expose state, then get out of CSS’s way.
