Scroll-triggered, not scroll-linked — no animation-timeline here. Each panel reads its own --live-visible-ratio: the fraction of it on screen, counting 0 → 100% as it enters view. One IntersectionObserver drives all four — watch them track independently, with no scroll handler.

03 / range

One value, many readers

Bind the range source to the container. It finds the inner <input> but writes --live-value and --live-value-pct on the container — so the gauge ring, the counted number, and the slider itself all read the same value (custom properties inherit down). Drag or press ← → — at either end a @container style() query flips a discrete min / max state that var() can’t express.

04 / style query

Discrete state, whole rules

var() can interpolate a number, but it can’t turn 3 into the word “High” or switch a rule block on at one exact value. That’s @container style(). Step through five levels — drag or press ← → — and each integer flips a different block of CSS on: the label’s text, its colour, the lit stop. One bound value, zero JS branches.

Four demos, one idea: expose state, then get out of CSS’s way.
